《Project: Memory.log》第二章白盒开发待办表 4.0

文档定位：
本文件用于把第二章现有设计拆成可执行的白盒开发任务。
目标不是写理想化路线图，而是给当前 Godot 原型一个明确的实现顺序。

--------------------------------------------------
一、当前工程现状判断
--------------------------------------------------

从当前项目看，已经具备以下基础：

1. 主角 A/B 层切换
2. A/B 层不同物理参数
3. A 层射击
4. B 层大锤基础动作
5. 子弹 -> B 层风道
6. 单向平台下跳
7. 基础敌人追击与护盾怪
8. 白盒竞技场触发器
9. 基础测试房 test_room

这意味着第二章白盒开发，不该再从“有没有核心机制”开始，
而应该从“怎么把现有机制收成真正能玩的章节原型”开始。

--------------------------------------------------
二、开发优先级总原则
--------------------------------------------------

优先级排序只按一个标准：
什么最能最快验证第二章根玩法。

因此优先级应为：
1. 战斗主循环验证
2. 关卡教学路线验证
3. 敌人与动作分工验证
4. 打击感和演出补强
5. 复杂资源与高级怪

不要反过来做。
如果在战斗主循环还没站稳时就先扩充大量怪物或写很多演出，后面返工会非常重。

--------------------------------------------------
三、P0：必须优先完成
--------------------------------------------------

P0-1
任务：
把 test_room 重做成“前 10 分钟白盒路线”

目标：
不再只是散放怪和收集品，而是真正承载：
- A 层基础教学
- Page-Lock Guard 错层痛感
- 切层权限解锁战
- A -> B 风道教学
- B -> A 单体切回教学
- 第一场综合战

落地建议：
当前建议改为：
优先按 `docs/AI持续开发/08_Godot项目AI友好化改造方案.txt` 建立 chapter2 新结构。
在结构骨架建立前，允许继续把 `test_room.tscn` 作为临时白盒验证场景；
但不应再把它视作唯一长期承载场景。

--------------------------------------------------

P0-2
任务：
实现 A 层“踢怪 / 强制挂起到 B 层”

目标：
验证“去正确层处理敌人”这一第二章核心教学。

必要功能：
1. A 层命中确认
2. 目标层级归属变化
3. 目标切入 B 层后状态保留
4. 玩家可立刻切入 B 层继续处理

这是前 10 分钟第一次降维打击的前置机制。
如果这条不成立，第二章最基础的“送去正确层处理”就无法真机验证。

--------------------------------------------------

P0-3
任务：
实现 B 层“锤攻击 + 切层键”带回单体敌人

目标：
验证最基础的 B -> A 转场动作。

必要功能：
1. 命中确认
2. 带目标切层
3. 回到 A 层后的连段延续
4. 敌人状态保留

这是当前最关键的新机制之一。
如果这个不好用，第二章后面的分流设计都会站不住。

--------------------------------------------------

P0-4
任务：
确定一只真正可测试的“B 层优势怪”

目标：
先做出一只能清楚证明“这怪在 B 层更好打”的新样板怪。

推荐：
优先白盒实现 `Page-Lock Guard` 的新版本，而不是继续沿用现在的护盾怪逻辑小修小补。

原因：
第二章的核心判断，需要一个非常清楚的样板敌人，不应该只靠临时拼凑。

--------------------------------------------------

P0-5
任务：
确定一只真正可测试的“A 层优势怪”

目标：
先做出一只能够稳定证明“这怪在 B 层不好处理，应切回 A 层”的样板怪。

推荐：
优先白盒实现 `Front Hound` 或 `Anchor Lurker`。

原因：
只有两类样板怪都成立，玩家才会真正学会“B 层稳杀 vs 切回 A 层”的分流。

--------------------------------------------------
四、P1：主循环成形后马上补
--------------------------------------------------

P1-1
任务：
实现 B 层重击切层的群体带回 / AOE 改场原型

目标：
验证这招是否真的在做“局面重排”，而不是“伤害更高的大锤”。

白盒要求：
不需要华丽表现，但必须有明确的：
- 命中范围
- 带回判定
- A 层第二段范围攻击

--------------------------------------------------

P1-2
任务：
实现 A 层控制技能切层 -> B 层护盾 / 防护收益

目标：
解决“玩家进入 B 层容易吃亏”的问题，并让 A 层控制技拥有明确战术身份。

白盒要求：
先只验证功能，不急着做复杂视觉。

--------------------------------------------------

P1-3
任务：
把 A 层炸弹切层做成真正的异化布置技

目标：
避免 A 层只有枪 -> 风道一种切层思路。

白盒要求：
先确定一个功能方向即可：
- 持续危险区
- 拉扯敌人
- 限位
- 爆炸扩散

不要一开始试图全做。

--------------------------------------------------

P1-4
任务：
实现第一版功能怪

推荐：
`Swap Priest` 或 `Privilege Leech`

目标：
让战斗从“只看单体”升级到“看优先级”。

--------------------------------------------------
五、P2：系统站稳后补强
--------------------------------------------------

P2-1
任务：
实现 B 层控制技切层的位移 / 抢位原型

说明：
这招当前设计仍不稳定，不应过早投入太多。

策略：
先做最低成本原型。
如果测试后用途窄、破坏空间判断或不好学，应直接重做，不硬保。

--------------------------------------------------

P2-2
任务：
实现节奏怪 / 场控怪组合

推荐组合：
- Fragment Swarm
- Sector Miner

目标：
测试 B 层重击切层在群战中的必要性。

--------------------------------------------------

P2-3
任务：
加入中型精英 `Cache Colossus`

目标：
检验中后段战斗循环是否成立。

说明：
它不适合最早做，因为在基础循环不稳时，大型怪只会放大问题。

--------------------------------------------------
八、脚本与系统任务拆分
--------------------------------------------------

S-1
任务：
为玩家动作系统增加“动作类型 -> 切层派生类型”的统一接口

目标：
不要把每一种切层动作都写成临时特判。

--------------------------------------------------

S-2
任务：
为敌人增加“当前层优势 / 回层后状态变化 / 是否适合切回”的统一数据结构

目标：
让敌人设计逐步数据化，而不是全靠写死逻辑。

--------------------------------------------------

S-3
任务：
为 B -> A 带怪机制建立统一处理流程

至少应包括：
- 被带回对象记录
- 位置同步
- 状态同步
- 伤害与硬直保留
- 群体带回兼容

--------------------------------------------------

S-4
任务：
为切层技能建立统一成本系统

建议先做最轻版：
- 普通切层消耗
- 重切层消耗
- 控制 / 防御切层消耗

先验证节奏，不急着做复杂经济。

--------------------------------------------------
九、测试顺序建议
--------------------------------------------------

正确测试顺序：

1. 只测一只 B 层优势怪 + 普通锤击切回
2. 再测一只 A 层优势怪 + 普通锤击切回
3. 再测枪 -> 风道
4. 再测重击切层群体带回
5. 再测功能怪混编
6. 最后才测前 10 分钟完整路线

不要一上来就测全部。
那样你根本分不清是动作错了、怪错了、地形错了，还是玩家教学顺序错了。

--------------------------------------------------
十、当前最值得立刻动手的 5 件事
--------------------------------------------------

1. 先做 chapter2 结构改造与场景骨架
2. 建立最小自动测试基础设施
3. 再重做 `test_room.tscn` 或迁移其内容为前 10 分钟白盒结构
4. 再实现 A 层踢怪 / 强制挂起到 B 层
5. 再实现 B 层“锤攻击 + 切层键”带回单体敌人
6. 新做一只 B 层优势怪原型与一只 A 层优势怪原型

只要这六件事做出来，第二章前 10 分钟白盒就能从“文档成立”进入“真机可判断”阶段。

--------------------------------------------------
十一、总结
--------------------------------------------------

第二章现在最缺的不是更多点子，而是第一套真正能跑通的白盒主循环。

这份待办表的核心意思只有一句：
先把“正确层处理敌人 + B 层稳杀 / 切回分流 + 六种动作至少跑通一半”做出来，
然后再谈更复杂的资源、更多怪和更华丽的演出。
