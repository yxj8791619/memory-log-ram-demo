《Project: Memory.log》第二章总设计文档 4.0

文档定位：
本文件用于替代《第二章的玩法初步-双层地图》的旧版讨论稿，作为第二章当前有效的总设计文档。
本版重点不是继续发散点子，而是收束第二章的根玩法、战斗决策、资源流向与关卡节奏。

--------------------------------------------------
一、章节定位
--------------------------------------------------

章节代号：
Sector_RAM / 垃圾回收协议 (Garbage Collection)

章节目标：
让玩家第一次真正理解“切层”不是视觉切换，而是改变战斗解法、移动路线和风险收益结构的底层权限。

章节体验关键词：
长卷轴推进、双层决策、主动转场、前后台切换、风险换收益、从压抑到降维打击

当前版本的总原则：
1. 第二章的根玩法，仍然是“双层怪物 + 玩家主动切层决策”。
2. A/B 两层不是镜像对称系统，而是职责明确、非对称分工的双战场。
3. 允许跨层连招，但不能让它反客为主，把 B 层降格成纯连招动画中的过场层。
4. 玩家在每次切层时，做的应该是战术判断，而不是解读复杂规则说明。

--------------------------------------------------
二、第二章核心规则
--------------------------------------------------

第二章的核心一句话：
玩家通过主动切换 A/B 两层，在正确的层处理正确的怪；并在合适的时机，利用不同类型的切层派生把局势转回自己更有利的战场。

这句话拆开后，对应四条基础规则：

1. 玩家可以主动 TAB 切层。
切层是免费、无缝、即时的，是本章最核心的权限。

2. 怪物在 A/B 两层的强弱、行为和解法不同。
“这只怪应该在哪层处理”是玩家做决策的第一层。

3. A 层拥有主动改变敌人层级的核心手段。
也就是主动“踢怪 / 强制挂起”技能。
这意味着 A 层是发起维度控制、建立局势的前台战场。

4. A/B 两层的核心动作都允许触发切层派生。
统一规则是“动作都能切”，不统一的是切层后的作用、成本、风险和收益。
这意味着系统更易学，但平衡必须依靠动作分工、资源消耗和怪物回层风险来约束。

--------------------------------------------------
三、A/B 两层职责分工
--------------------------------------------------

🔴 Layer A：活跃内存 / 前台进程

定位：
前台执行层、主动发起层、资源建立层。

视觉听觉：
高对比霓虹青色，画面干净、节奏快、读屏清晰。BGM 清脆、锐利、有推进感。

物理手感：
重力正常，移动和跳跃更利落，适合走位、风筝、跑图和建立火力覆盖。

战斗职责：
1. 用枪械与机动压制战场。
2. 主动发起“踢怪 / 强制挂起”，改变敌人的层级归属。
3. 积攒资源，为进入 B 层做准备。
4. 制造切层后的残留结果，例如风道、特殊爆炸、护盾等非目标型跨层转化效果。

🟣 Layer B：虚拟内存 / 后台挂起

定位：
高收益处理层、爆发层、破甲层、风险兑现层。

视觉听觉：
暗色、低频、带拖影与噪点。强调“后台”“迟滞”“正在被系统硬读写”的氛围。

物理手感：
重力更轻，移动略粘，允许悬空处理、压重击和高爆发近战。

战斗职责：
1. 处理那些“在 B 层打收益更高”的敌人。
2. 用近战重击、破甲、处决和压制来兑现 A 层建立的优势。
3. 用不同类型的切层派生完成单体转场、群体转场、位移抢位或战场塑形。

关键结论：
A 层负责建立优势与改变层级。
B 层负责兑现优势与高收益处理。

--------------------------------------------------
四、切层连招的最终定义
--------------------------------------------------

第二章允许跨层连招，但不采用“敌人映射残留在另一层，玩家过去追着映射打完，再回来找本体”的设计。
原因很明确：
这种设计解释复杂、操作累赘、节奏碎裂，而且会强迫玩家反复确认本体与映射归属。

第二章采用的是“非对称切层派生”：

1. A -> B 的跨层技能
以非指定目标的 AOE、功能、Buff、战场塑形类效果为主。
这类技能通常不要求把某一只敌人一起带层。
它们的作用是把 A 层动作结果，转化成 B 层的战术资本。

当前适合保留的方向：
- A 层射击 -> B 层风道
- A 层炸弹 -> B 层异化引爆
- A 层控制波 -> B 层护盾或防御收益

2. B -> A 的跨层连招
以命中具体敌人为主。
一旦触发，就把该敌人直接带回 A 层继续战斗，不经过映射、副本、残影本体分离等中间层解释。

补充规则：
第二章的统一原则是，A 层 3 种基础动作和 B 层 3 种基础动作都可以触发切层派生。
玩家学习的是“所有核心动作都能切”，而不是记忆“哪些能切、哪些不能切”。
真正区分它们的，是切层后的战术角色不同：
- A 层切层更偏向转化、布置、护身、塑形
- B 层切层更偏向带怪回层、转场收尾、群体改写、位移抢位

这条规则的结果是：
- 玩家在 B 层对单体连招时，逻辑足够明确。
- 连招中断、未击杀、继续输出都天然成立。
- 不会出现“切过去打了一半，打完还得切回原层接着找本体”的累赘体验。

--------------------------------------------------
五、B 层击杀与切层连招的分工
--------------------------------------------------

这是第二章战斗决策的核心补充规则。

玩家在 B 层处理一只怪时，不是永远都应该用切层连招，也不是永远都应该留在 B 层打死。
正确决策，取决于两个问题：

1. 这只怪回到 A 层后，会不会重新变强或更难处理？
2. 我现在是否需要把战斗节奏主动转回 A 层？

因此，B 层存在两套合法打法：

第一种：B 层稳杀
适用情况：
- 这只怪在 B 层是主要处理层。
- 它如果回到 A 层会重新变强、变快、变烦或重新进入优势形态。
- 当前战场还没清干净，玩家不想过早把压力带回 A 层。

玩家决策：
不使用切层连招，直接在 B 层用普通近战和重击把它打死。
击杀后，玩家再自行 TAB 回 A 层。

第二种：B -> A 切层连招
适用情况：
- 玩家现在正想回到 A 层继续推进。
- 这只怪即使被带回 A 层，短时间内仍能控住或有机会收掉。
- 切层连招本身可以带来更帅、更高伤害、更高收益的收尾或转场。

玩家决策：
用 B 层切层派生把敌人带回 A 层。
如果这一套能杀，直接完成高收益击杀。
如果这一套不能杀，战斗就自然在 A 层继续，不需要额外解释。

关键补充：
“切层连招必须能斩杀”不是硬规则。
只要系统上允许把敌人从 B 带回 A，战斗就可以继续。
这时真正的风险判断就出现了：
如果这只怪在 A 层更强，而玩家又判断这波切招打不死，那么他就应该明确选择不切，留在 B 层稳妥处理完毕。

再补充一类特殊情况：
如果玩家当前在 B 层，而这只怪实际上在 A 层更好处理，那么玩家没有必要强行留在 B 层硬打。
由于 B 层唯一能改变敌人层级归属的方式，就是 B -> A 的切层连招，
所以这类怪物的标准解法就是：
先在 B 层命中、启动切层连招，把目标带回 A 层，再在 A 层完成主要处理。

因此，B -> A 切层连招有三种角色：
1. 收尾击杀
2. 主动转场
3. 把“不适合在 B 层处理的怪”送回 A 层

--------------------------------------------------
六、怪物设计原则
--------------------------------------------------

怪物设计的重点，不再是“它能不能被切层”，而是以下三个问题：

1. 它在哪一层更强？
2. 它在哪一层更适合被处理？
3. 它是否值得被提前带回另一层？

第二章的怪物应该围绕这三个问题做分类。

分类一：A 层优势怪
定义：
它在 B 层处理效率不高，或者 B 层无法稳定压制。
设计用途：
逼迫玩家在 B 层命中后，尽快想办法把它带回 A 层。

分类二：B 层优势怪
定义：
它在 A 层难缠、低效、刮痧、危险，去 B 层处理明显更好。
设计用途：
构成第二章最基础的层级决策。
这类怪通常不鼓励玩家过早用切层连招把它带回 A 层，除非玩家明确判断能杀或愿意承担风险。

分类三：可转场怪
定义：
它在两层都有战斗空间，但切层后表现会明显变化。
设计用途：
专门服务于“B 层稳杀”与“B -> A 切招”两种打法分流。
这类怪是第二章连招爽点的重要承载体。

分类四：功能怪 / 支援怪
定义：
它本身的血量和伤害不一定高，但它会影响其它敌人的层级玩法。
例如：
- 赋予免踢
- 提供护盾
- 让某层危险度升高
- 压缩玩家在某层的停留空间

这类怪物的存在意义是：
让玩家的切层判断不是“单体最优输出”，而是“战场优先级调度”。

--------------------------------------------------
七、资源设计原则
--------------------------------------------------

资源系统服务于切层决策，不单独喧宾夺主。

当前资源设计的目标不是做复杂经济，而是强化一句话：
A 层负责建立，B 层负责兑现。

资源应满足以下方向：

1. 玩家在 A 层更容易积攒进入 B 层所需的攻击资本。
2. 玩家在 B 层进行高收益处理时，能够明显感受到“兑现”的爽感。
3. B -> A 切层连招可以拥有额外收益，但不能高到把“B 层稳杀”彻底挤死。
4. “所有动作都能切”是统一学习规则，但不同动作的切层收益不能同质化。

适合的收益方向：
- 更高伤害
- 更高资源返还
- 更好的转场位置
- 对 A 层下一轮开战更有利的局面

不适合的方向：
- 简单粗暴的“B 层一律伤害 x2”
- 切层连招成为所有情况的唯一最优解
- 玩家只算倍率，不看怪物与场面

--------------------------------------------------
八、技能设计筛选结论
--------------------------------------------------

以下是现阶段应该保留和推进的技能方向。

第二章当前推荐保留“六种基础动作都可切层派生”的统一框架：

A 层三类：
1. 普通攻击切层：转化型
2. 算力攻击切层：布置型
3. 控制技能切层：防御 / 辅助型

B 层三类：
1. 普通锤击切层：单体带回型
2. 重击切层：群体带回 / AOE 改写型
3. 控制技能切层：位移 / 抢位 / 开路型

应该保留的方向：
1. A 层射击 -> B 层风道
理由：
这是非常标准的“前台动作结果转化为后台战术资本”。

2. A 层炸弹 -> B 层异化爆炸
理由：
适合做范围控制、开场布置、地形清理或逼位。

3. A 层控制波 -> B 层护盾 / 防御收益
理由：
这是非目标型跨层收益，逻辑顺、解释稳。

4. B 层普通锤击切层 -> 命中后带单体敌人回 A 层
理由：
适合做单体处决、转场点杀与接机枪收尾，是 B -> A 最直观的切层模板。

5. B 层重击切层 -> 带回群体敌人，并在 A 层触发第二段 AOE
理由：
适合做群体转场、局面重排和高成本高回报的爆发玩法。

6. B 层控制技能切层 -> 位移 / 抢位 / 贴近目标
理由：
可以作为位置技或开路技，但当前用途仍偏窄，适合保留为实验项继续验证。

应该谨慎限制的方向：
1. “所有攻击后摇都能保留到切层后的层级”
风险：
系统解释成本高，容易泛化成万能切层取消和万能连段胶水。

2. 所有动作都能切层，但切层后的收益如果过于同质
风险：
会让玩家只记住“切了就赚”，最终把 B 层降格为纯过场层。

3. 投技拉敌人途中切层、瞬移到敌人位置这类强位移跨层技能
风险：
极容易破坏空间判断，抢走“双层怪物解法”的主导权。

--------------------------------------------------
九、关卡与教学节奏
--------------------------------------------------

第二章前段体验，仍然延续旧版文档中的“先抑后扬”结构，但教学重点要更明确。

第一阶段：只有 A 层的普通跑打
目标：
建立玩家对横版射击的基础预期。

第二阶段：遇到在 A 层难处理的敌人
目标：
让玩家明确感受到“错误层处理敌人”是低效和痛苦的。

第三阶段：解锁踢怪与切层
目标：
让玩家理解 A 层负责发起层级控制，B 层负责高收益处理。

第四阶段：教学 A -> B 非目标型跨层收益
目标：
让玩家理解切层不仅是追击，还能把战场塑形。

第五阶段：教学 B -> A 切层连招
目标：
让玩家理解所有 B 层核心动作都能切层，但普通锤击、重击和控制技切层后的作用完全不同。
同时让玩家理解，这不是必须斩杀的处决技，而是主动把战斗带回 A 层的转场技。

第六阶段：混合战场
目标：
让玩家同时做三层判断：
1. 哪只怪该在哪层打
2. 现在该不该主动切层
3. B 层这只怪是稳杀，还是切回 A 层继续打

--------------------------------------------------
十、现阶段的开发落点
--------------------------------------------------

当前白盒开发不应该再继续发散更多切层招式，而应优先验证以下内容：

1. 让玩家在实战中真正感受到：
“这只怪在 A 层打很亏，在 B 层打很赚。”

2. 验证 B -> A 切层连招的手感与风险：
玩家能否自然判断：
"- 这波我要不要切回 A？”
"- 这怪带回 A 后会不会出事？”

3. 验证“所有动作都能切层”是否真的更易学，而不是更容易滥用。

4. 验证 A -> B 的非目标型收益是否真的改变战场，而不是只有视觉新鲜感。

5. 验证 B 层不会因为切层连招过强，而退化成纯连招过场层。

如果这四件事成立，第二章的根玩法才算真正站住。

--------------------------------------------------
十一、总结
--------------------------------------------------

第二章不是“两个层都能自由追同一只敌人的实时跨层连招游戏”。
第二章的正确方向是：

1. 双层怪物弱点与解法，仍然是玩家决策核心。
2. A 层负责发起、建立、踢怪、积攒和塑形。
3. B 层负责高收益处理、破甲、爆发和少量主动转场。
4. A -> B 以非目标型战术转化为主。
5. B -> A 以命中具体敌人的切层连招为主，但不同动作的切回效果应严格分工。
6. 玩家在 B 层面对敌人时，真正的判断不是“能不能切”，而是：
   这只怪该不该留在 B 层稳杀，还是该被我带回 A 层继续处理。

这套结构，既保住了第二章最早确立的双层决策核心，也保住了后来加入的切层连招爽点，但不会让两者互相冲突。
